# Issue Types

This file defines the issue types available for your project.
Copy this file to `./track/types.md` to customize type descriptions for intelligent type assignment.

## FEATURE

New functionality that doesn't exist yet. Use for adding capabilities, screens, APIs, or workflows that users don't currently have access to.

## BUG

Something is broken or not working as expected. Use when existing functionality fails, produces wrong results, or doesn't match documented behavior.

## ENHANCEMENT

Improvement to existing functionality. Use for optimizations, refactoring, better UX, performance improvements, or extending capabilities that already exist.

## TASK

Technical work, chores, and maintenance. Use for infrastructure changes, dependency updates, configuration, documentation, or housekeeping that doesn't directly add features or fix bugs.
